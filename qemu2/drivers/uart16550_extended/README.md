# Драйвер контроллера UART 16550A

## Описание

Данный проект содержит учебно-дипломную реализацию драйвера контроллера UART,
совместимого с программной моделью 16550A. Драйвер предназначен для работы в
операционной системе Linux и взаимодействует с устройством через MMIO-регистры.

Проект используется в составе системы-на-кристалле, где UART реализован либо в
QEMU-модели, либо в RTL на FPGA. Основная задача драйвера — показать полный
маршрут взаимодействия:

```text
Пользовательская программа
        ↓
/dev/uart16550_demo
        ↓
Linux kernel module
        ↓
MMIO-регистры UART
        ↓
QEMU/RTL модель контроллера UART
```

## Что реализовано

В драйвере реализованы следующие возможности:

- загрузка и выгрузка как Linux kernel module;
- создание символьного устройства `/dev/uart16550_demo`;
- отображение MMIO-области устройства через `ioremap`;
- доступ к UART-регистрам через `readb`, `writeb`, `readl`, `writel`;
- базовая инициализация UART в режиме 8N1;
- настройка делителя скорости через `DLL/DLM` и бит `DLAB`;
- включение и выключение loopback-режима через `MCR`;
- сброс FIFO через `FCR`;
- передача данных через регистр `THR`;
- приём данных через регистр `RBR`;
- проверка состояния устройства через `LSR`;
- поддержка `read`, `write`, `poll`, `ioctl`;
- чтение статистических регистров модели `TX_COUNT`, `RX_COUNT`, `ERR_COUNT`, `IRQ_COUNT`;
- расширенное логирование через параметр `debug_level`;
- встроенный self-test через `ioctl`;
- набор пользовательских тестов.

## Аппаратная модель

В текущей версии используется APB-wrapper, в котором регистры UART разнесены как
32-битные слова. Поэтому регистр с индексом `N` расположен по адресу:

```text
base + N * 4
```

Текущий базовый адрес по умолчанию:

```text
0xff210000
```

```text
uart16550_demo.c        # драйвер Linux kernel-space

test_uart16550_demo.c   # расширенный тестовый набор
Makefile                # сборка модуля и тестов
run_tests.sh            # автоматический запуск сборки и тестов
README.md               # описание проекта
```

## Сборка

```bash
make all
```

Сборка состоит из двух частей:

1. сборка kernel module `uart16550_demo.ko`;
2. сборка пользовательского теста `test_uart16550_demo`.

## Загрузка драйвера

```bash
sudo insmod uart16550_demo.ko base_addr=0xff210000 loopback=1 debug_level=2
```

Проверка:

```bash
dmesg | tail
ls -l /dev/uart16550_demo
```

## Запуск тестов

```bash
sudo ./test_uart16550_demo --verbose
```

Также можно использовать автоматический скрипт:

```bash
./run_tests.sh
```

## Выгрузка драйвера

```bash
sudo rmmod uart16550_demo
```

## Параметры модуля

| Параметр | Назначение |
|---|---|
| `base_addr` | базовый MMIO-адрес UART |
| `region_size` | размер MMIO-области |
| `loopback` | включение loopback-режима при инициализации |
| `debug_level` | уровень логирования |
| `tx_timeout_us` | таймаут ожидания готовности передатчика |
| `rx_timeout_us` | таймаут ожидания данных на приём |
| `default_divisor` | значение делителя UART по умолчанию |

## Уровни логирования

| Уровень | Описание |
|---|---|
| `0` | только ошибки |
| `1` | основные информационные сообщения |
| `2` | подробные сообщения драйвера |
| `3` | трассировка MMIO-чтений и записей |

Для максимально подробного вывода:

```bash
sudo insmod uart16550_demo.ko base_addr=0xff210000 loopback=1 debug_level=3
```

## ioctl-интерфейс

Драйвер поддерживает следующие команды:

| Команда | Назначение |
|---|---|
| `UART_IOCTL_INIT_BASIC` | базовая инициализация UART |
| `UART_IOCTL_GET_STATS` | чтение аппаратных счётчиков модели |
| `UART_IOCTL_GET_REGS` | чтение основных регистров UART |
| `UART_IOCTL_SET_DIVISOR` | настройка делителя скорости |
| `UART_IOCTL_SET_LOOPBACK` | включение/выключение loopback |
| `UART_IOCTL_RESET_STATS` | сброс статистических регистров |
| `UART_IOCTL_RESET_FIFO` | сброс FIFO |
| `UART_IOCTL_GET_CONFIG` | получение состояния драйвера |
| `UART_IOCTL_SELF_TEST` | встроенный тест передачи/приёма |

## Тестовый набор

Тестовая программа проверяет:

- инициализацию UART;
- чтение регистров;
- чтение конфигурации драйвера;
- настройку делителя;
- включение loopback;
- сброс статистики;
- сброс FIFO;
- запись строки;
- рост `TX_COUNT`;
- приём байта в loopback-режиме;
- работу `poll`;
- неблокирующее чтение;
- встроенный self-test;
- стресс-тест на 1000 байт;
- финальную статистику.

## Ограничения

При **аппаратном** loopback через MCR UART на FPGA:

- Если к линии **RX FPGA** всё же подключён **TX USB‑TTL**, возможны ошибки RX; драйвер **перед каждой отправкой байта во внутреннем loopback опустошает RX FIFO** из аппаратных «хвостов» (до 64 символов), чтобы не смешивать их с локальным echo. Если мешает внешняя активная логика всё равно, отключите **TX переходника** от RX FPGA **на время** тестов.

Данный драйвер является учебной реализацией и не заменяет стандартный драйвер
Linux `serial8250`. В текущей версии не реализованы:

- полноценная интеграция с `tty/serial_core`;
- обработка аппаратных прерываний через `request_irq`;
- DMA;
- полноценная поддержка `termios`.

При этом драйвер демонстрирует ключевую для дипломной работы функциональность:
взаимодействие Linux kernel-space с UART-контроллером через MMIO-регистры.

## Назначение в дипломной работе

Драйвер показывает программный уровень взаимодействия операционной системы с
контроллером UART. В рамках общего проекта он связывает Linux с UART-моделью,
которая может быть реализована в QEMU или в RTL на FPGA.
